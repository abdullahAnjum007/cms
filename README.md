Building Systems That Solve Real Problems.
🌱 The Beginning

This project marks a chapter in my developer journey.

Not a tutorial app.
Not a copy-paste clone.
But a system built with intention.

CMS (Complaint Management System) started as a simple idea:

“What if complaint handling in small organizations could be structured, trackable, and transparent?”

This is where I began building systems instead of just screens.

🎯 Vision

To create a clean, scalable, role-based mobile application that:

Organizes complaints efficiently

Connects users with technicians

Gives admins complete system visibility

Feels modern, simple, and practical

This is not overloaded with features.
It’s structured to grow.

🧠 The Concept

CMS revolves around 3 core roles, each with a clear responsibility.

👤 Complaint Poster (User)

Create complaints

Upload evidence using mobile camera

Track status in real-time

Maintain complaint history

Access personal dashboard

🛠 Technician

View assigned tasks

Manage complaint queue

Update complaint status

Track work progress

Maintain resolution workflow

🛡 Admin

View all complaints in the system

Monitor technicians

Manage users

Access analytics & reports

Full control and system transparency
